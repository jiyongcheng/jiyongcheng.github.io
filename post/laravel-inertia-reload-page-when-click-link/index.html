How to Prevent Page Reloads in Laravel Inertia with Vue.js and PrimeVue

One common issue developers face when working with Laravel Inertia and Vue.js is page reloads when clicking links in the menu. This typically happens when the Inertia <Link> component is not used correctly. In this article, we’ll explore why this happens and how to fix it with a practical example.

Why Do Page Reloads Happen?

When using Inertia.js to build single-page applications (SPAs), the navigation should be smooth and seamless without full page reloads. However, if you use standard HTML anchor (<a>) tags instead of Inertia’s <Link> component, Inertia cannot intercept the link clicks, resulting in full page reloads.

How to Fix Page Reloads

To prevent page reloads, you need to replace standard anchor tags with Inertia’s <Link> component. Here’s a step-by-step guide to achieve this in a Laravel Inertia project using Vue.js and PrimeVue components.

Step 1: Install and Set Up Inertia.js

If you haven’t already, install Inertia.js and its dependencies.

  1. Install the server-side adapter:
1

composer require inertiajs/inertia-laravel
  1. Install the client-side adapter:
1

npm install @inertiajs/inertia @inertiajs/inertia-vue3
  1. Add Inertia middleware in app/Http/Kernel.php:
1
2
3
4

'web' => [
    \App\Http\Middleware\HandleInertiaRequests::class,
    // other middleware...
],
  1. Create the middleware:
1

php artisan inertia:middleware

createInertiaApp({
    resolve: name => require(`./Pages/${name}`),
    setup({ el, app, props, plugin }) {
        createApp({ render: () => h(app, props) })
            .use(plugin)
            .mount(el)
    },
});

InertiaProgress.init();

import { Link } from '@inertiajs/vue3';

<template>
    <li :class="{ 'layout-root-menuitem': root, 'active-menuitem': isActiveMenu }">
        <div v-if="root && item.visible !== false" class="layout-menuitem-root-text">{{ item.label }}</div>
        <Link v-if="(!item.to || item.items) && item.visible !== false" :href="item.url" @click="itemClick($event, item, index)" :class="item.class" :target="item.target" tabindex="0">
            <i :class="item.icon" class="layout-menuitem-icon"></i>
            <span class="layout-menuitem-text">{{ item.label }}</span>
            <i class="pi pi-fw pi-angle-down layout-submenu-toggler" v-if="item.items"></i>
        </Link>
        <Link v-if="item.to && !item.items && item.visible !== false" :href="item.to"  :class="[item.class, { 'active-route': $page.url === item.to }]">
            <i :class="item.icon" class="layout-menuitem-icon"></i>
            <span class="layout-menuitem-text">{{ item.label }}</span>
            <i class="pi pi-fw pi-angle-down layout-submenu-toggler" v-if="item.items"></i>
        </Link>
        <Transition v-if="item.items && item.visible !== false" name="layout-submenu">
            <ul v-show="root ? true : isActiveMenu" class="layout-submenu">
                <app-menu-item v-for="(child, i) in item.items" :key="child" :index="i" :item="child" :parentItemKey="itemKey" :root="false"></app-menu-item>
            </ul>
        </Transition>
    </li>
</template>

Conclusion

By replacing standard anchor tags with Inertia’s component, you can ensure smooth navigation without full page reloads in your Laravel Inertia application. This approach leverages the power of Inertia.js to create a seamless SPA experience, enhancing user experience and performance.

Implement these changes in your project, and enjoy a smoother, more responsive application!
